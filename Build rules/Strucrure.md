## 1. Разметка устройства.

Цель:

Быстро и красиво доводить задумку до пользователя 3 видами действий: \

1.1 Создание.

- По задумке приложения дать его устройство и заложить все необходимые \
  свойства
- Определить наилучшие пути и способы создания, поддержки и развития - \
  раскат на стендах и тестирование, деплой, исправление и доработка, \
  доставка, масштабирование, приспособления и правила для \
  отказоустойчивости, наблюдаемости.
- Рассчитать стоимость создания, поддержки и развития приложения \
  на основании его технических показателей.
- Суметь Построить, снять и сверить рассчеты и способы Создания

1.2 Поддержка.

- Удерживать приложение в установленных границах технических показателей.
- При выходе за них разработать ряд новых решений для Поддержки

1.3 Развитие

- По снятым рассчетам и новым задумкам и решениям имеющихся трудностей \
  при Создании и Поддержки обновить их способы и пустить новый виток Создания.

## 2. Правила:

Разбиваем приложение на уровни подходом от общего к частному. От целого \
пользовательского приложения до отдельных микросервисов и строк кода.
Каждый уровень делим одинаково на 3 раздела: Описание, Показатели, \
Действия.

Описание. Здесь описываем уровень с разделением на Внутреннее \
устройство и Внешнее поведение. Внешнее поведение показывают то, \
с какими составляющими узлами одного уровня общается \
Текущий уровень и как это происходит. \
Внутреннее устройство показывает то, как общаются между собой \
составляющие нижнего уровня и то, каким образом это определяет Внешнее \
поведение
Важное отличие: Внешнее поведение не показывают то, как текущий \
уровень влияет на Внешнее поведение Верхнего Уровня.

Показатели. \

Здесь даются технические показатели, выступающие как меры Внешнего \
поведения (МВП), внутреннего устройства (МВУ), а также показатели \
стоимости Создания, Поддержки и Развития уровня (ПСУ - показтели \
стоимости уровня). МВП как функция от МВУ. ПСУ как функция МВП.
Тут же даны все расчеты и пояснения

Действия. \

Здесь перечислены применения лучших способов Создания, Поддержки и Развития \
для уровня. Их перечень в разделе 1.1



## Уровень 1. Целое приложение

### 1.1 Описание

#### 1.1.1 Внешние поведение.


MVP площадки Искуственных Переговорщиков. Можно создавать личности. \
Можно общаться с личностями, которых сам создал. \
Вход и авторизация через отпечаток браузера. \
Общение происходит в классическом в наше время интерфейсе чата. \

Разработчик отслеживает состояние приложения и управляет им \
через всевозможные панели управления и дашборды, покрытые \
всевозможными показателями.
Также разраб имеет доступ к панели управления моделями для их \
всевозможной конфигурации - от подугрузки чекпоинта до запуска \
пайплайна дообучения. Сейчас только возможность выбора модели. \
Остальное как заглушка

#### 1.1.2 Внутреннее устройство.

Сейчас создание личности только промтом-предустановкой.\
Это шаблоны, которые можно выбирать (например юрист, шутник).\
В будущем можно настраивать личность через свойства психики во \
всех тонкостях её устройства.
Память личности и беседы с ней в VAD графах и векторах.\
Заложена простая эмпатия. 
Есть поле ввода текста, список личностей, раскатывающееся поле \
бесед с ними. Окно создания личностей. Ответ личности выкатывается \
на лету - через streaming tokens без ожидания полного ответа.

Имеется панель управления, где можно регулировать задействие \
мошностей при масштабировании. Тут можно поставить автонастройки \
для kuber либо самостоятельно выводить на машины инстансы приложения \
(например при поавной обкатке обновления, когда мы постепенно \
проверяем способность фичи работать в бою под нагрузкой.)

Есть панель управления моделями. Тут их можно подгружать и назначать \
на различные задачи.

#### 1.2.1 Хранение и доставка

Инстанс может быть развернут docker compose, а грубое \
кратное расширение микросервисов через docker swarm. \
Если требуется развертка отдельных микросервисов с различной \
кратностью и прочие тонкости приспособления к нагрузке - тут всецело \
полагаемся на kubernetes. Обе возможности доступны из командной \
строки
